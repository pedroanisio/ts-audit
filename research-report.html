1. Introduction

The proliferation of safety-critical systems across automotive, aerospace, medical, and industrial domains has created an urgent need for principled methods to compare and translate safety requirements across different regulatory frameworks. Each domain has developed its own safety integrity level taxonomy—SIL (IEC 61508), ASIL (ISO 26262), DAL (DO-178C)—reflecting domain-specific concerns about failure modes, operational environments, and acceptable risk levels.

This report synthesizes research on mathematical foundations for cross-standard requirements analysis, organizing knowledge in a graph structure that captures formal relationships between concepts, theories, and applications. We draw upon lattice theory, formal concept analysis, and category theory to establish rigorous foundations for integrity level comparison.

The research is organized around 5 primary investigation threads:

  1. Mathematical Foundations for Requirements Decomposition: How do we rigorously decompose, compare, and compose requirements across different logical frameworks?
  2. Integrity Classification Theory: How do we map impact assessments to integrity levels to engineering constraints in a domain-agnostic way?
  3. Safety Standards Research: How do we extract, formalize, and map requirements across safety-critical domains?
  4. Neuro-Symbolic AI / Universal Knowledge Compilation: How do we combine LLM capabilities with formal knowledge representations for reliable, verifiable AI?
  5. Modal, Epistemic, and Causal Reasoning: How do we formalize multi-agent knowledge, belief revision, temporal properties, and causal inference?

The challenge of AI safety in this context is aptly summarized by leading researchers:

"In some sense, hallucination is all LLMs do. They are dream machines... It's only when the dreams go into deemed factually incorrect territory that we label it a hallucination." — Andrej Karpathy, OpenAI, Tesla AI

2. Mathematical Frameworks

We employ 6 complementary mathematical frameworks to address different aspects of requirements decomposition and cross-standard mapping. Each framework provides specific capabilities while acknowledging inherent limitations.

2.1. Category Theory (CT)

Mathematical meta-framework underlying institution theory for specification composition

Complexity: Abstract - implementation dependent

Application: Compositional semantics and verification

2.2. Description Logics (DL)

Formal foundation of OWL providing precise semantics for requirement subsumption and equivalence

Complexity: N2ExpTime-complete for SROIQ, ExpTime-complete for ALC

Application: Ontology reasoning and semantic validation

Key Operations:

  • Subsumption
  • Classification
  • Consistency checking

2.3. Feature Models (FM)

Well-established framework for variability modeling with propositional semantics

Complexity: NP-complete (SAT-based)

Application: Configuration and variability

2.4. Formal Concept Analysis (FCA)

Structural backbone for decomposing standards into atomic requirements and discovering relationships

Complexity: Polynomial for lattice construction

Application: Requirements decomposition and clustering

Key Operations:

  • Derivation operators
  • Implication discovery
  • Canonical basis

2.5. Galois Connections (GC)

Rigorous machinery for abstraction-concretization relationships

Complexity: Depends on poset structure

Application: Best approximation reasoning

2.6. Institution Theory (IT)

Mathematically cleanest abstraction for comparing standards in different logical frameworks

Complexity: Depends on underlying logics

Application: Cross-framework translation

Key Operations:

  • Comorphisms
  • Grothendieck construction

3. Formal Theory

This section presents the formal definitions and theorems that constitute the mathematical foundation of the integrity classification framework.

3.1. Definitions

Definition 1 (Integrity Lattice) [ICT]
$$L = (E, ≤, ⊥, ⊤)$$

E is finite set of integrity levels, ≤ is total order, ⊥ is bottom, ⊤ is top

Note: Since E is totally ordered, forms a CHAIN: join(a,b) = max(a,b), meet(a,b) = min(a,b)

Definition 2 (Impact Space) [ICT]
$$I = (D₁ × D₂ × ... × Dₙ, ⊕)$$

Each Dᵢ is finite ordered set of impact dimensions, ⊕ is aggregation function

Definition 3 (Classification Functor) [ICT]
$$Φ: I → L$$

Monotonic function mapping impact assessments to integrity levels

Definition 4 (Constraint Space) [ICT]
$$C = (C, ≤c)$$

C is set of engineering constraint bundles, ≤c is partial order (more stringent than)

Definition 5 (Constraint Derivation Functor) [ICT]
$$Ψ: L → C$$

Monotonic function mapping integrity levels to constraint bundles

Definition 6 (Universal Ordinal Space) [ICT]
$$U = ([0, 1], ≤, 0, 1)$$

The canonical bounded chain into which all integrity lattices embed

Definition 7 (Normalization Functor) [ICT]
$$ν: L → U$$

ν(ℓⱼ) = j / (k - 1) for j-th level in lattice with k levels

Definition 8 (System Composition) [ICT]
$$⊙$$

For subsystems s₁...sₙ with levels ℓ₁...ℓₙ, system-level ℓ_sys determined by composition operator

3.2. Theorems

Theorem 1 (Composition Theorem) [ICT]

The composition Ψ ∘ Φ: I → C is monotonic

Theorem 2 (Galois Connection Theorem) [ICT]

If Φ has a right adjoint Φᴿ: L → I, then (Φ, Φᴿ) forms a Galois connection

Theorem 3 (Universal Property Theorem) [ICT]

For lattices L₁, L₂ with normalizations ν₁, ν₂: a ≤ᵤ b ⟺ ν₁(a) ≤ ν₂(b) is reflexive and transitive

Theorem 4 (Decomposition Theorem) [ICT]

If system S with requirement ℓ_sys decomposes into independent S₁...Sₙ

5. Research Hypotheses

The following hypotheses guide ongoing research in cross-standard requirements analysis and AI safety verification. Each hypothesis is formulated to be empirically testable or formally provable.

Hypothesis H1: Feedback Loop Necessity Hypothesis [Architectural Necessity]

For LLM-generated outputs requiring conformance to a formal ontology O, iterative validation with feedback produces significantly higher conformance rates than single-pass generation

Status: supported

Hypothesis H10: Guarantee-Performance Incompatibility Hypothesis [Empirical Reality]

Systems achieving high precision on validation do so through strategic abstention, trading recall for reliability

Hypothesis H11: Claim-Performance Divergence Hypothesis [Empirical Reality]

Commercial AI validation claims systematically overstate empirical performance by a factor of 2-10x

Hypothesis H12: Syntactic-Semantic Validation Dichotomy [Empirical Reality]

Current techniques provide strong guarantees for syntactic correctness but only soft constraints for semantic accuracy

Hypothesis H13: Guarantee Transformation Hypothesis [Achievability]

Iterative ontology validation transforms system guarantee from infeasibility to epistemic transparency

Hypothesis H14: Layered Mitigation Efficacy Hypothesis [Achievability]

Layered mitigation strategies achieve meaningful but bounded risk reduction

Hypothesis H15: Domain Restriction Efficacy Hypothesis [Achievability]

Formal validation achieves highest efficacy in domains with enumerable rule space, complete formal specification, closed-world semantics, and static knowledge base

Hypothesis H16: Knowledge Acquisition Bottleneck Hypothesis [Knowledge Acquisition]

Even if verification were tractable, knowledge formalization rate is bounded by scarcity of dual-expertise practitioners

Hypothesis H17: Incompleteness Unknowability Hypothesis [Knowledge Acquisition]

For any knowledge base KB, determining the location and degree of incompleteness is itself an unsolved problem

Hypothesis H2: Dual-Layer Validation Hypothesis [Architectural Necessity]

Effective ontology validation requires two complementary formal mechanisms operating jointly: SHACL (structural, closed-world) and OWL Reasoner (logical, open-world)

Status: supported

Hypothesis H3: Tripartite Feedback Architecture Hypothesis [Architectural Necessity]

Effective ontology-constrained LLM output requires three jointly necessary components: Violation Detection (V), Explanation Generation (E), and Constrained Regeneration (R)

Status: supported

Hypothesis H4: Parsing Bottleneck Hypothesis [Bottleneck]

The NL→RDF semantic parsing step is the primary source of validation failure, independent of ontology quality or reasoning completeness

Hypothesis H5: Coverage Ceiling Hypothesis [Bottleneck]

Validation accuracy is upper-bounded by ontology coverage κ, where empirically κ ≈ 0.04–0.30 for even best-formalized domains

Hypothesis H6: Expressiveness-Decidability Tradeoff Hypothesis [Bottleneck]

There exists a fundamental tradeoff between ontology expressiveness and computational tractability that imposes hard limits on validatable constraint types

Hypothesis H7: Convergence Stratification Hypothesis [Bottleneck]

Convergence behavior is stratified by constraint complexity class C

Hypothesis H8: Verification Intractability Hypothesis [Theoretical Impossibility]

Complete formal verification of neural network outputs is computationally intractable for production-scale LLMs

Hypothesis H9: Architectural Hallucination Hypothesis [Theoretical Impossibility]

Hallucination is architecturally intrinsic to LLMs, not a correctable defect

Hypothesis UKC-H1: Knowledge Compilation Hypothesis [UKC]

For any domain D with finite, expressible knowledge K_D, there exists a formal representation F(K_D) in some logic L such that entailment queries Q can be decided by a mechanical procedure

Hypothesis UKC-H2: NL-to-Logic Parsing Hypothesis [UKC]

For LLM output O in natural language NL, there exists a parsing function P: NL → L that maps O to logical statements with accuracy A ≥ threshold T sufficient for downstream verification

Hypothesis UKC-H3: Feedback Loop Efficacy Hypothesis [UKC]

Let H₀ be baseline hallucination rate. A formal validation system V with feedback loop F produces outputs with rate H₁ such that H₁/H₀ ≤ ε for acceptably small ε

Hypothesis UKC-H4: Computational Tractability Hypothesis [UKC]
